Stash Commands: 
---------------
its a Git command that temporarily saves your uncommitted changes (both staged and unstaged),
so you can switch branches, pull updates, or work on something else without committing incomplete work.

git stash                    Stash changes
git stash save "message"     Stash with description
git stash list               List all stashes
git stash show stash@{n}     Show changes
git stash apply stash@{n}    Apply stash
git stash pop stash@{n}      Apply and remove stash
git stash drop stash@{n}     Delete stash
git stash clear              Remove all stashes

Note: any command without using stash@{n} will work with n as default position

Reset & Revert Commits:
---------------
Reset (removes commits completely): Move head pointer to a specific commit
git log --oneline            Show prev commits with hashes
git log --graph --oneline --all  show branch graph
    -To make git graph as alias name: git config --global alias.graph "log --graph --oneline --all --decorate --color"
git reflog		     Show all commits with hashes
git show hash                Commit details
git diff hash1..hash2         Compare commits
git reset --hard hash        Reset to commit
    - "--hard" Removes current commit changes and get hash changes
git push origin main --force Force push changes

also can use HEAD instead of hash:

Fast Backward: Move head pointer backward 
    - We can use HEAD~N to get last N commit instead of using hash
Fast Forward: move head pointer forward
    - We can use HEAD@{N} to get last N commit instead of using hash


Revert (safe undo with new commit):
git revert hash              Revert single commit
git revert old^..new         Revert range of commits

How to contribute with Open Source projects:

1. Fork the Repository (on GitHub)
2. Clone Your Fork Locally
3. Add the Original Repo as "Upstream" (to sync later)
git remote add upstreamName https://github.com/original-owner/repo.git

-Verify remotes:
git remote -v
# Should show:
# origin    https://github.com/your-username/repo.git (fetch/push)
# upstream  https://github.com/original-owner/repo.git (fetch/push)
4. Create a New Branch and Make Changes & Commit then Push
5. Sync with Upstream (Optional, Avoid Conflicts)
-Fetch latest changes from the original repo:
git fetch upstreamName
git merge upstreamName/main  # or `git rebase upstream/main`
Resolve conflicts if any (see conflict resolution guide).
6. Create a Pull Request then submit it

Tagging & Releases:

Semantic Versioning (SemVer) is a widely adopted versioning scheme for software that helps convey meaning about the underlying changes in each release.
It follows the format: MAJOR.MINOR.PATCH (e.g., 2.5.1).

Semantic Versioning Rules:
-MAJOR version – Incremented for backward-incompatible changes (breaking changes).
-Example: 1.0.0 → 2.0.0 (API changes that require user action).
-Resets MINOR and PATCH to 0.
-MINOR version – Incremented for backward-compatible new features.
-Example: 1.2.3 → 1.3.0 (new functionality in a backward-compatible way).
-Resets PATCH to 0.
-PATCH version – Incremented for backward-compatible bug fixes.
-Example: 1.2.3 → 1.2.4 (bug fixes, no new features).

It happens if your remote URL is set to HTTPS instead of SSH. Check with:

┃ git remote -v

If it shows https://..., switch it back to SSH:

┃ git remote set-url origin git@github.com:USERNAME/REPO.git

After that, pushing will use your SSH key again instead of asking for
username/password.
